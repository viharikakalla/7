8051 INSTRUCTION SET

ARITHEMATIC INSTRUCTIONS

  • It mostly performs 8-bit operations .
  • The result of the op-code is stored in A-Register.
  • For some special operations B-Register is used .

⭐️ ADDITION OPERATION :
1. ADD A,#25

  • This instruction adds A-register with the 8-bit DATA , and stores the result in A-register.
    2.ADD A,REG. (R7 - R0)
  • This instruction adds A-register with data present in the specified registers.
    3. ADD A,25H
  • This instruction adds A-register with the content present in the given address.
    4.ADD A,@(R0/R1)
  • Here only R0 and R1 registers are used .
  • This instruction adds A-register with the contents located in R0/R1 .
    5. ADDC A,#25H
    ADDC - Add with Carry
  • This instruction adds A-register with more than 8-bit data(16,32) this is done by splitting 16/32 - bit data into parts . Lower bits and Higher bits .
  • Carry Flag holds the carry of this operation.
  • 8-bitResult is stored in A-register and the remaining 1-bit is holds carry in CARRY FLAG .
    6. ADDC A,R0
  • This instruction adds A-register with data present in the specified registers and holds carry in CARRY FLAG .
    7. ADDC A,25H
  • This instruction adds A-register with the content present in the given address and holds carry in CARRY FLAG.
    8. ADDC A,@R0/R1
  • This instruction adds A-register with the contents located in R0/R1 and holds carry in CARRY FLAG.

⭐️ SUBTRACTION OPERATION :
SUBB - Subtract with Borrow

  • To do normal subteraction we have to first clear the CARRY FLAG . Else CF will interfear in the operation .
    9. SUBB A,#25
  • This instruction subtracts A-register with the 8-bit DATA , stores the result in A-register and holds carry in CARRY FLAG .
    10. SUBB A,REG. (R7 - R0)
  • This instruction subtracts A-register with data present in the specified registers and holds carry in CARRY FLAG .
    11. SUBB A,25H
  • This instruction subtracts A-register with the content present in the given address and holds carry in CARRY FLAG .
    12. SUBB A,@(R0/R1)
  • Here only R0 and R1 registers are used .
  • This instruction subtracts A-register with the contents located in R0/R1 and holds carry in CARRY FLAG .

⭐️ INCREMENT OPERATION :
13. INC A

  • Increments the value of A-register .
    14. INC R0
  • Increments the value of given register .
    15. INC 25H
  • Increments the contents of location 25 .
    16. INC @R0
  • Increments the contents ponted by location R0 .
    17. INC DPTR
  • Increments the value of DPTR.
  • Requires two cycles to perform operation.
  • DPTR only increment is possiable.

⭐️ DECREMENT OPERATION :
18. DEC A

  • Decrements the value of A-register .
    19. DEC R0
  • Decrements the value of given register .
    20. DEC 25H
  • Decrements the contents of location 25 .
    21. DEC @R0
  • Decrements the contents ponted by location R0 .

⭐️ MULTIPLICATION OPERATION :

  • Result of this instruction is 16-bit
    22. MUL AB
  • This instruction multiplies the B-register with A-register and the result is stored in A-register and B-register.
  • A - register stores Lower byte .
  • B - register stores Higher byte.

⭐️ DIVISION OPERATION :
23. DIV AB

  • This instruction divides the B-register with A-register and the result is stored in A-register and B-register.
  • Where A is DIVIDENT and B is DIVISOR
  • A - register stores Quotient .
  • B - register stores Remainder.

24. DA A
DA A - Decimal Adjust After Addition

  • This instruction is performrd only after addition .
    Used only after adding 2 decimal numbers .
graph TB A[[DA A operation ]] --> 1[HN > 9 / AC = 1]; A --> 2[LN > 9 / CF = 1]; 1 ---> a(then add 6 to Higher nibble); 2 ---> b(then add 6 to Higher nibble);

DA A instruction follows the following steps :

  1. Addition.
  2. Obtain the result (hexadecimal form).
  3. convert hexadecimal result into decimal form .

⭐️ Rotate Instructions :
1.RL A

  • This instruction rotates all 8-bits towards left within A-register.
  • Here carry flag just holds the copied value of MSB.
CF <-- MSB LSB

1.RR A

  • This instruction rotates all 8-bits towards Right within A-register.
  • Here carry flag just holds the copied value of MSB.
MSB LSB --> CF

3.RLC A

  • This instruction rotates all 8-bits towards left along with CF in A-register.
  • Here carry flag value is also rotated.
CF MSB LSB

4.RRC A

  • This instruction rotates all 8-bits towards right along with CF in A-register.
  • Here carry flag value is also rotated.
MSB LSB CF

BOOLEAN INSTRUCTIONS

BOOLEAN instructions are BIT instructions i.e, they perform bitwise opertions

  • For bit operations CARRY FLAG is the ACCUMULATOR
  • For any operation the given data, first have to be stored in CARRY FLAG , only then it performs the operation. After performing the operation the result is stored in Carry flag. We can also move the result stored in CF to other specified register .
  • Bits can either be a Carry Flag. Or bits can be a General Bit .
    • SFR Bits
    • PSW Bits
    • Bit Addressable (internal RAM)

⭐️ Carry Flag (C)
1. SETB C
sets the carry flag to 1 (CF = 1)
2. CLR C
clears the carry flag to 0 (CF = 0)
3. CPL C
Complements the Carry flag .

⭐️ Any General Bits (b)
1. SETB b
sets the bit to 1 (b = 1).
2. CLR b
clears the bit to 0 (b = 0).
3. CPL b
Complements the bit .

⭐️ MOVES
1. MOV C,b
This instruction moves the data from b to C.And stores the result in C.
2. MOV b,C
This instruction moves the data from C to b.And stores the result in b.

⭐️ LOGIC
1. ANL C,b
This instruction performs AND operation of C and b . The result is stored in C.
2. ANL C,/b
This instruction performs AND operation of C and completed b.The result is stored in C

3. ORL C,b
This instruction performs OR operation of C and b . The result is stored in C.
4. ORL C,b
This instruction performs OR operation of C and completed b . The result is stored in C.

⭐️ CALL :-

  • "CALL" is used to go to SUBROUTINE.
  • Before going to subroutine , the "PC" stores the subroutine address in it .
  • The PC also stores the "RETURN ADDRESS"(RA) as soon as the SUBROUTINE is called.
  • The PC pushes the RA on to the STACK to store untill the CALL completes its TASK.
  • As soon as the task is completed , the stack pops the RA to the PC .
